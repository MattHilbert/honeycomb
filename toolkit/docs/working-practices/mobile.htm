After reviewing the last 12 months analytics, we can see less than 5% of our site visitors are using a mobile device. Therefore the decision has been made to not support low screen resolutions.

We also review our mobile approach during design stages, as it may be there is a low overhead to introducing some mobile support.

We realise mobile is hugely important, and we need to review our mobile support regularly.

Simple things we can do

  • Keep bandwidth to a minimum
  • Add touch support – 44px x 44px minimum tap size
  • Don’t rely on :hover, or mouseover events to get to content
  • Keep popups stand-alone – they cannot send data to the parent (e.g. binoculars)
  • Avoid scrolling content areas – It can be achieved in Mobile Safari using touch events and CSS transitions but it’s not easy
  • (TO BE DECIDED) Optimise styles for small screens with @media queries & viewport meta

Browser Testing

We support modern mobile browsers Opera & Safari.

  • Opera Mobile & Opera Mini
  • Mobile Safari

You can test Opera Mobile from your desktop machine – using Opera Mobile Emulator.

You can also introspect Opera mobile inside a phone, tablet or TV using Opera Dragonfly.

The iPhone SDK comes with Mobile Safari but is mac only, there are iPads around the office and a lot of people have iPhones (they might let you borrow one). Make sure error reporting is enabled in Safari settings.

(If you’re keen, here’s a pretty long list of mobile emulators: http://www.mobilexweb.com/emulators )

Media queries

Deliver specific styles to different devices, they can test for things like:

  • screen or print
  • width and height
  • orientation â€“ is mobile device in landscape or portrait mode?
  • resolution

You can optimise styles for different devices

                @media only screen and (max-device-width: 480px) {
                  /* hide header and footer on small screen devices */
                  #header, #footer { display: none }
                }
                @media only screen and (max-width:800px) {
                  /* linearize navigation under 800px */
                  nav, nav li, nav a { float: none; width: 100% }
                }

The viewport meta tag is also needed to instruct a mobile device to keep the viewport to the amount of pixels it has on the screen i.e. prevent it from zooming out like a desktop version.

Great article on viewport and media queries http://dev.opera.com/articles/view/an-introduction-to-meta-viewport-and-viewport/
